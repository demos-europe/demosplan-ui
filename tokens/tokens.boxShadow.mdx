# Box Shadow Tokens

Box shadow tokens provide consistent elevation effects across the UI. These tokens help create visual hierarchy and depth in the interface.

## Usage

### Tailwind Usage

Box shadow tokens can be applied directly in HTML with Tailwind utility classes:

```html
<!-- Basic shadow -->
<div class="shadow-sm p-4 bg-surface">
  Card with standard shadow
</div>

<!-- Elevated element -->
<div class="shadow-md p-4 bg-surface">
  Elevated card
</div>

<!-- Modal -->
<div class="shadow-lg p-6 bg-surface">
  Modal content
</div>

<!-- Sidebar -->
<aside class="shadow-xl p-4 bg-surface h-screen">
  Sidebar content
</aside>

<!-- Hover effects -->
<button class="shadow-sm hover:shadow-md active:shadow-lg">
  Button with dynamic shadows
</button>
```

### SCSS Usage

```scss
// Import box shadow tokens
@import '@demos-europe/demosplan-ui/tokens/scss/boxShadow';

// Use in your styles
.card {
  box-shadow: $dp-box-shadow;
}

.modal {
  box-shadow: $dp-box-shadow-lg;
}

.button {
  box-shadow: $dp-box-shadow;

  &:hover {
    box-shadow: $dp-box-shadow-md;
  }
}
```

## Guidelines

- Use shadows consistently to convey elevation in the interface
- Higher elevation (larger shadows) should be used for elements that appear "closer" to the user
- Combine shadows with slight background color changes for more distinct separation
- Use shadows sparingly to avoid visual clutter
