# Font Size Tokens

Font size tokens define the typography scale for demosplan-ui. They include both numerical scale tokens and semantic tokens for different UI contexts.

## Usage

### Tailwind Usage

Font size tokens can be used directly in HTML with Tailwind utility classes:

```html
<!-- Basic font sizes -->
<p class="text-base">Default text size (16px)</p>
<p class="text-xs">Very small text (10px)</p>
<p class="text-lg">Larger text (18px)</p>

<!-- Semantic heading sizes -->
<h1 class="text-h1">Heading Level 1</h1>
<h2 class="text-h2">Heading Level 2</h2>
<h3 class="text-h3">Heading Level 3</h3>

<!-- Special cases -->
<div class="text-hero">Hero Text</div>

<!-- With line height -->
<p class="text-h2 leading-h2">Heading with correct line height</p>
```

### Tailwind v4 Line Height Pairing

In Tailwind v4, font sizes with associated line heights use a paired variable format:

```html
<!-- This will apply both the font size and appropriate line height -->
<h1 class="text-h1">Heading with paired line height</h1>
```

### SCSS Usage

```scss
// Import font size tokens
@import '@demos-europe/demosplan-ui/tokens/scss/fontSize';

// Basic usage
.element {
  font-size: $dp-font-size-medium;
}

// With inuit-font-size mixin (for line height)
.heading {
  @include inuit-font-size($dp-font-size-h1, 1.3);
}
```

## Usage Guidelines

- **Heading tokens** (`h1`-`h6`): Use for content structure in long-form text
- **UI tokens** (`smallest`, `small`, `medium`, etc.): Use for interface elements like buttons, tabs, etc.
- **Special tokens** (`header`, `hero`): Use sparingly for very specific branded sections

<Notice content={
  `Avoid mixing heading tokens with UI tokens in the same context. For user interfaces, use UI tokens; for content structure, use heading tokens.`
} />
