# Border Radius Tokens

Border radius tokens define the consistent rounding of corners throughout the UI. These tokens help create a cohesive visual language and maintain consistent component styling.

## Available Border Radius Values

  headers={['Description', 'Tailwind Class', 'SCSS Variable', 'Use Case']}

    ["None", "rounded-none", "$dp-rounded-none", "Elements with sharp corners"],

    ["Small (Default)", "rounded-sm", "$dp-rounded-sm", "Default for most UI elements, form inputs, buttons"],

  ]}

## Usage

### Tailwind Usage

Border radius tokens can be applied directly in HTML with Tailwind utility classes:

```html
<!-- Basic usage -->
<button class="rounded-sm">Default button</button>
<div class="rounded-md">Card with medium radius</div>
<span class="rounded-full px-2 py-1">Pill label</span>

<!-- Directional rounding -->
<div class="rounded-t-lg">Rounded top corners only</div>
<div class="rounded-r-md">Rounded right corners only</div>
<div class="rounded-bl-sm">Rounded bottom-left corner only</div>

<!-- Responsive variants -->
<div class="rounded-sm md:rounded-lg">
  Changes radius at different breakpoints
</div>

<!-- Combining with other properties -->
<button class="rounded-sm bg-interactive text-on-dark p-2">
  Styled button
</button>
```

### SCSS Usage

```scss
// Import border radius tokens
@import '@demos-europe/demosplan-ui/tokens/scss/rounded';

// Use in your styles
.button {
  border-radius: $dp-rounded-sm;
}

.card {
  border-radius: $dp-rounded-md;
}

.badge {
  border-radius: $dp-rounded-full;
}
```

## Guidelines

- Use `rounded-sm` as the default for most UI elements
- Maintain consistent rounding within similar component types
- Use `rounded-xs` for nested elements within already rounded containers
- Apply `rounded-full` for pills and circular elements
- When in doubt, choose the smaller radius option
